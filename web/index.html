How it works

1
Receiver clicks "Receive"

Browser generates RSA-2048 keypair. Private key stays in memory. Public key displayed for sharing.

2
Receiver shares public key

Copy the arc1pk_... key and send it via Slack, email, SMS - any channel.

3
Sender encrypts & sends

Secret is encrypted in browser with receiver's public key, then sent to relay server as an encrypted blob.

4
Receiver decrypts

Relay forwards encrypted blob to receiver. Browser decrypts with private key. Keys are destroyed.

What does the relay do?

Relay Server

Acts as a real-time pipe between sender and receiver. Cannot decrypt - only receiver has the private key. No data stored, just forwarded instantly.

Encryption RSA-OAEP, 2048-bit, SHA-256
Key Format arc1pk_ + base64url(SPKI)
Key Hash SHA-256, first 16 bytes, base64url
Max Payload ~190 bytes (RSA-OAEP limit)
Transfer Real-time pipe, no storage
Retry Auto-retry for 2 min if needed
